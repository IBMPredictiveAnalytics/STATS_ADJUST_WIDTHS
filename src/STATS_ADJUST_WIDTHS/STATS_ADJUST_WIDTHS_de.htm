Breiten von Zeichenfolgevariablen dateiübergreifend anpassen

Wenn Dateien zusammengeführt werden, die Zeichenfolgevariablen enthalten, müssen die übereinstimmenden Variablen dieselbe Breite aufweisen. Mit dieser Prozedur werden die Breiten mehrerer Dateien oder Datasets synchronisiert.

Das Dialogfeld greift auf die aktive Datei sowie auf eine externe Datei zu. Mithilfe der Syntax kann eine ganze Reihe von Dateien gleichzeitig verarbeitet werden.

►Wählen Sie zur Ausführung dieser Prozedur folgende Optionen aus den Menüs aus:

 Daten
  Daten>Variablenbreiten anpassen...

Zu synchronisierende Datei Wählen Sie eine Datei aus, die mit der aktiven Datei synchronisiert werden soll.

• Wenn die Datei bereits offen ist, wird sie erneut geöffnet, wobei alle nicht gespeicherten Änderungen verloren gehen.

• Damit dieser Befehl ausgeführt werden kann, muss die aktive Datei über einen Datasetnamen verfügen.

Anzupassende Variablen Wählen Sie die (Zeichenfolge-) Variablen aus, deren Breiten angepasst werden sollen

• Eine ausgewählte Variable muss nicht in jeder Datei vorhanden sein.

Größenanpassungsregel Wählen Sie aus, in welcher Weise die Breiten angepasst werden sollen. Die Anpassung wird für jede Variable separat durchgeführt. Die Variablenbreiten werden maximal an die breiteste Instanz der Variable angepasst etc. Bei Auswahl von 'Erste' wird die Zielbreite der der Variablen im aktiven Dataset entsprechen.

• Wenn das resultierende Feld zu klein ist, um alle vorliegenden Werte aufzunehmen, werden diese Werte abgeschnitten.

Maximale Größe Die Größenregeln werden durch das hier angegebene Maximum beschränkt.

Präfix für Datasetnamen Wenn eine Datei geöffnet wird, wird ihr ein Datasetname bestehend aus dem Präfix und einer sequenziellen ganzen Zahl zugewiesen.

• Wenn dieser Datasetname bereits verwendet wird, wird er dieser Datei neu zugewiesen. Das andere Dataset wird geschlossen. Noch nicht gespeicherte Änderungen gehen dabei verloren.

Die Änderungen werden mit dem Befehl ALTER TYPE vorgenommen. Davon betroffene Variablen werden in der Ausgabe dieses Befehls aufgelistet.

Speichern

Die geänderten Dateien bleiben standardmäßig geöffnet und ungespeichert. Das Unterdialogfeld 'Speichern' bietet eine einfache Möglichkeit zum Speichern.

Nach Anpassungen geänderte Dateien erneut speichern Wählen Sie dieses Feld aus, wenn die verarbeiteten Dateien gespeichert werden sollen.

• Damit das aktive Dataset gespeichert wird, muss unten ein Verzeichnis angegeben werden.

• Wenn an der Datei keine Änderungen vorgenommen wurden, wird sie auch nicht erneut gespeichert.

Vorhandene Datei überschreiben Wählen Sie dieses Feld aus, wenn eine vorhandene Datei durch die geänderte Version überschrieben werden soll.

Suffix an Dateinamensstamm anhängen Wenn eine Datei gespeichert wird, kann ein Suffix an den Stamm des Dateinamens angehängt werden. Beispiel: Aus der Angabe '_angepasst' könnte der Dateiname 'meineDaten_angepasst.sav' gebildet werden

Datei nach dem Speichern schließen Wählen Sie dieses Feld aus, wenn die Datei nach ihrer Verarbeitung automatisch geschlossen werden soll. Dies ist kontraproduktiv, wenn die Datei nicht außerdem noch gespeichert wird, wobei die aktive Datei jedoch nie geschlossen wird.

Dateien in Verzeichnis schreiben Wählen Sie ein Verzeichnis aus, in das die Dateien geschrieben werden sollen. Wenn keine Angabe gemacht wird, werden die Dateien an ihrem ursprünglichen Speicherort gespeichert.

• Damit die aktive Datei auf die Festplatte gespeichert wird, muss ein Verzeichnis angegeben werden. Beim Speichern erhält sie den Namen 'dsname.sav', wobei 'dsname' für den Datasetnamen steht. Es wird kein Suffix angehängt.

Weitere Funktionen

Mithilfe der Syntax können Sie mit mehr als zwei Eingabedateien oder Datasets arbeiten, wobei die aktive Datei nicht eingeschlossen werden muss. Der Befehl funktioniert auch bei nur einer einzelnen Datei.

Wenn Sie ein Verzeichnis angeben, die letzte Ebene jedoch nicht vorhanden ist, wird sie von diesem Befehl erstellt.

Ausführen
STATS ADJUST WIDTHS /HELP.
Sie zeigen damit die vollständige Syntax an.

Hinweis: Die ursprüngliche Version dieses Befehls speicherte auch dann Dateien erneut, wenn sie nicht geändert wurden. Obwohl dieses Verhalten unter bestimmten Bedingungen komfortabel ist, wurde es zugunsten einer besseren Handhabbarkeit einer sehr großen Anzahl von Dateien geändert.

Anforderungen

Für diesen Befehl sind Python Essentials und der Erweiterungsbefehl STATS ADJUST WIDTHS von der SPSS Community-Website (www.ibm.com/developerworks/spssdevcentral) erforderlich.
