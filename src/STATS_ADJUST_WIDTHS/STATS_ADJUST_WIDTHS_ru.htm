Согласовать длины строковых переменных среди файлов

При объединении файлов, содержащих строковые переменные, у совпадающих переменных должна быть одинаковая длина. Эта процедура синхронизует длины в собрании файлов или наборов данных.

В этом диалоговом окне работают с активным файлом и одним внешним файлом. Правильно используя синтаксис, можно одновременно обрабатывать несколько файлов.

►Чтобы запустить эту процедуру, выберите в меню:

 Данные
  Данные>Согласовать длины переменных...

Файл для синхронизации Выберите файл, который нужно синхронизовать с активным файлом.

• Если этот файл уже открыт, он будет открыт повторно, а все несохраненные данные будут потеряны.

• Чтобы можно было запустить эту команду, у активного файла должно быть имя набора данных.

Переменные для согласования Выберите строковые переменные, длины которых нужно согласовать

• Выбранная переменная не должна присутствовать в каждом файле.

Правила согласования размера Выберите, как нужно согласовать длины. Согласование производится отдельно для каждой переменной. При выборе Максимум длины переменных будут согласованы по самому длинному экземпляру переменной. Если выбрать опцию Первый, длиной назначения будет длина переменной в активном наборе данных.

• Если полученное поле окажется слишком маленьким, чтобы содержать все свои значения, эти значения будут усекаться.

Максимальный размер Этим заданным максимумом ограничиваются правила размеров.

Префикс для имен наборов данных Когда файл открыт, ему назначается имя набора данных, состоящее из префикса и последующего целого числа.

• Если это имя набора данных уже используется, оно переназначается данному файлу, а другой набор данных закрывается. Ранее не сохраненные изменения теряются.

Чтобы произвести изменения, используется команда ALTER TYPE. О затронутых переменных будет сообщено в выводе этой команды.

Сохранить

По умолчанию измененные файлы остаются открытыми и не сохраняются. Для удобства функциональная возможность сохранения предоставляется в диалоговом подокне Сохранить.

В случае изменений повторно сохранить файл после согласований Включите этот переключатель, чтобы сохранить обработанные файлы.

• Активный набор данных не будет сохранен, пока ниже не будет задан каталог.

• Если файл не был изменен, он повторно не сохраняется.

Перезаписать существующий файл Включите этот переключатель, чтобы перезаписать существующий файл измененной версией.

Присоединить суффикс к корню имени файла При сохранении файла к корню имени файла будет присоединен суффикс. Например, если задать для суффикса значение _adjusted, результирующим именем файла будет mydata_adjusted.sav

Закрыть файл после сохранения Включите этот переключатель, чтобы автоматически закрыть файл после обработки. В этом нет смысла, если файл одновременно не сохраняется, а активный файл никогда не закрывается.

Записать файлы в каталог Выберите каталог, в который нужно записать файлы. Если такой каталог не задан, файлы будут сохраняться в исходном положении.

• Пока не задан каталог, активный файл не будет сохранен на диск. При сохранении файла он будет назван dsname.sav, где dsname - это имя набора данных. Суффикс не присоединяется.

Дополнительные возможности

Правильно используя синтаксис, можно работать более чем с двумя входными файлами или наборами данных, а активный файл включать необязательно. Эта команда будет работать также только с одиночным файлом.

Если вы указываете каталог, а его последний уровень не существует, он будет создан данной командой.

Выполните
STATS ADJUST WIDTHS /HELP.
чтобы вывести полный синтаксис.

Примечание: исходная версия этой команды повторно сохраняла бы файлы, даже если они не изменялись. Хотя это иногда удобно, такое поведение было изменено для улучшения работы с очень большим количеством файлов.

Технические требования

Для этой команды требуется Python Essentials и команда расширения STATS ADJUST WIDTHS с Веб-сайта сообщества SPSS (www.ibm.com/developerworks/spssdevcentral).
