Ajustar larguras de variáveis das sequências de caracteres entre arquivos

Ao mesclar arquivos contendo variáveis de sequência de caracteres, as variáveis correspondentes devem ter a mesma largura. Este procedimento sincroniza as larguras entre uma coleção de arquivos ou conjuntos de dados.

O diálogo opera no arquivo ativo mais um arquivo externo. Usando a sintaxe, é possível processar muitos arquivos de uma vez.

►Para executar este procedimento, nos menus, escolha:

Dados
  Dados>Ajustar larguras de variáveis...

Arquivo para sincronizar Selecione um arquivo para ser sincronizado com o arquivo ativo.

• Se o arquivo já estiver aberto, ele será reaberto e quaisquer mudanças não salvas serão perdidas.

• O arquivo ativo deve ter um nome de conjunto de dados para executar este comando.

Variáveis para ajustar Selecione as variáveis (sequência de caracteres) cujas larguras devem ser ajustadas

• Uma variável selecionada não precisa aparecer em cada arquivo.

Regra de ajuste de tamanho Escolha como as larguras devem ser ajustadas. O ajuste é feito separadamente para cada variável. Para máximo, as larguras das variáveis serão ajustadas para a instância mais larga da variável, e assim por diante. Se você escolher Primeiro, a largura de destino será aquela da variável no conjunto de dados ativo.

• Se o campo resultante for muito pequeno para conter todos os seus valores, esses valores serão truncados.

Tamanho máximo As regras de tamanho são restringidas por este máximo especificado.

Prefixo para nomes de conjuntos de dados Quando um arquivo é aberto, ele tem um nome de conjunto de dados designado que consiste no prefixo e em um número inteiro sequencial.

• Se esse nome do conjunto de dados já estiver em uso, ele será redesignado a este arquivo e o outro conjunto de dados será fechado. Mudanças não salvas anteriormente serão perdidas.

O comando ALTER TYPE é usado para fazer as mudanças. Variáveis afetadas serão relatadas na saída deste comando.

Salvar

Por padrão, os arquivos modificados serão deixados abertos e não salvos. O subdiálogo Salvar fornece a funcionalidade de salvamento por conveniência.

Salvar arquivo novamente após ajustes se modificado Marque esta caixa para salvar os arquivos processados.

• O conjunto de dados ativo não será salvo a menos que um diretório seja especificado abaixo.

• Se o arquivo não tiver sido modificado, ele não será salvo novamente.

Sobrescrever arquivo existente Marque esta caixa para sobrescrever um arquivo existente com a versão alterada.

Anexar sufixo à raiz do nome do arquivo Ao salvar um arquivo, um sufixo pode ser anexado à raiz do nome do arquivo. Por exemplo, especificar _adjusted pode resultar em um nome de arquivo mydata_adjusted.sav

Fechar arquivo após salvar Marque esta caixa para fechar automaticamente o arquivo após ser processado. Isto é contraproducente se o arquivo não for também salvo, no entanto, o arquivo ativo nunca é fechado.

Gravar arquivos no diretório Selecione um diretório no qual os arquivos devem ser gravados. Se nada for especificado, os arquivos serão salvos na localização original.

• O arquivo ativo não será salvo em disco a menos que um diretório seja especificado. Quando ele for salvo, o nome tem o formato dsname.sav, em que dsname é o nome do conjunto de dados. Nenhum sufixo é anexado.

Recursos adicionais

Usando sintaxe, é possível trabalhar com mais de dois arquivos ou conjuntos de dados de entrada, e o arquivo ativo não precisa ser incluído. O comando também trabalhará com somente um único arquivo.

Se você especificar um diretório e o último nível não existir, ele será criado por este comando.

Executar
STATS ADJUST WIDTHS /HELP.
para exibir a sintaxe completa.

Nota: a versão original deste comando salvaria os arquivos novamente mesmo se eles não tivessem sido modificados. Apesar de às vezes conveniente, esse comportamento foi alterado como parte de trabakhar melhor com um número muito grande de arquivos.

Requisitos

Este comando requer o Python Essentials e o comando de extensão STATS ADJUST WIDTHS do website SPSS Community (www.ibm.com/developerworks/spssdevcentral).
