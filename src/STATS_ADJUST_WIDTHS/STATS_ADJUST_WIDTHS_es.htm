Ajustar los anchos de variable de cadena en los archivos

Al fusionar archivos que contienen variables de cadena, las variables coincidentes deben tener el mismo ancho. Este procedimiento sincroniza los distintos anchos en una colección de archivos o conjuntos de datos.

El cuadro de diálogo funciona en el archivo activo más un archivo externo. Mediante la sintaxis, puede procesar muchos archivos a la vez.

►Para ejecutar este procedimiento, desde los menús seleccione:

Datos
  Datos>Ajustar los anchos de variable...

Archivo que se va a sincronizar Seleccione un archivo que se va a sincronizar con el archivo activo.

• Si el archivo ya está abierto, se volverá a abrir, y se perderán los cambios que no haya guardado.

• El archivo activo debe tener un nombre de conjunto de datos para poder ejecutar este comando.

Variables que se van a ajustar Seleccione las variables (cadenas) cuyo ancho debe ajustarse.

• No es necesario que una variable seleccionada aparezca en cada archivo.

Regla de ajuste de tamaño Elija cómo se van a ajustar los anchos. El ajuste se realiza de forma separada para cada variable. Como valor máximo, los anchos de variable se ajustarán a la instancia más amplia de la variable, y así sucesivamente. Si elige Primero, el ancho de destino será el de la variable del conjunto de datos activo.

• Si el campo resultante es demasiado pequeño para contener todos sus valores, estos valores se truncarán.

Tamaño máximo Las reglas sobre el tamaño están restringidas por este valor máximo especificado.

Prefijo para nombres de conjuntos de datos Al abrir un archivo, se asigna un nombre de conjunto de datos que se compone del prefijo y un entero secuencial.

• Si ese nombre de conjunto de datos ya se está utilizando, se reasigna a dicho archivo y el otro conjunto de datos se cierra. Se perderán los cambios que no se hayan guardado previamente.

El comando ALTER TYPE se utiliza para realizar los cambios. Las variables afectadas se notificarán en la salida de este comando.

Guardar

De manera predeterminada, los archivos modificados se quedan abiertos y no se guardan. Puede utilizar el subcuadro de diálogo Guardar para guardar los archivos cuando convenga.

Volver a guardar el archivo después de los ajustes si se ha modificado Marque esta casilla para guardar los archivos procesados.

• El conjunto de datos activo no se guardará a menos que se especifique un directorio más abajo.

• Si el archivo no se ha modificado, no se volverá a guardar.

Sobrescribir archivo existente Marque esta casilla para sobrescribir un archivo existente por la versión modificada.

Añadir sufijo a la raíz del nombre de archivo Al guardar un archivo, se puede añadir un sufijo a la raíz del nombre de archivo, por ejemplo, si especifica _ajustado, el nombre del archivo puede ser misDatos_ajustado.sav

Cerrar archivo después de guardar Marque esta casilla para cerrar automáticamente el archivo después de procesarlo. Esto puede ser contraproducente si el archivo no se ha guardado; no obstante, el archivo activo no se cierra nunca.

Escribir archivos en el directorio Seleccione un directorio en el que se van a grabar los archivos. Si no se especifica ninguno, los archivos se guardan en la ubicación original.

• El archivo activo no se guarda en el disco a menos que se especifique un directorio. Cuando se guarda el archivo, el nombre tiene el formato dsname.sav, en el que dsname es el nombre del conjunto de datos. No se añade ningún sufijo.

Características adicionales

Mediante la sintaxis, puede trabajar con más de dos archivos o conjuntos de datos de entrada, y no es necesario incluir el archivo activo. El comando también funciona con un único archivo.

Si especifica un directorio, y no existe el último nivel, este comando lo creará.

Ejecutar
STATS ADJUST WIDTHS /HELP.
para mostrar toda la sintaxis.

Nota: la versión original de este comando volvía a guardar los archivos aunque no se hubieran modificado. Aunque pudiera resultar conveniente, este comportamiento se modificó porque funciona mejor si el número de archivos es muy elevado.

Requisitos

Este comando requiere Python Essentials y el comando de extensión STATS ADJUST WIDTHS del sitio web de la Comunidad de SPSS (www.ibm.com/developerworks/spssdevcentral).
