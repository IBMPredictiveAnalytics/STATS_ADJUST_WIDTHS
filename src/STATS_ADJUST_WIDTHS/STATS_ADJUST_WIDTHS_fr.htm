Ajustement de la largeur des variables de chaîne dans les fichiers

Lorsque vous fusionnez des fichiers qui contiennent des variables de chaîne, les variables correspondantes doivent avoir la même largeur. Cette procédure permet de synchroniser les largeurs dans un ensemble de fichiers ou de jeux de données.

La boîte de dialogue fonctionne sur le fichier actif plus un fichier externe. A l'aide de la syntaxe, vous pouvez traiter plusieurs fichiers simultanément.

►Pour exécuter cette procédure, dans les menus, choisissez :

 Données
  Données>Ajuster la largeur des variables...

Fichier à synchroniser : Sélectionnez le fichier à synchroniser avec le fichier actif.

• Si le fichier est déjà ouvert, il est à nouveau ouvert et les modifications non enregistrées sont perdues.

• Le fichier actif doit comporter un nom de jeu de données pour pouvoir exécuter cette commande.

Variables à ajuster : Sélectionnez les variables (chaîne) dont la largeur a besoin d'être ajustée.

• La variable sélectionnée n'apparaît pas dans tous les fichiers.

Règle d'ajustement de la taille : Choisissez la méthode d'ajustement des largeurs. Ce réglage est effectué séparément pour chaque variable. La largeur maximale des variables sera ajustée à l'instance la plus large de la variable, et ainsi de suite. Si vous choisissez Première, la largeur cible sera celle de la variable dans le jeu de données actif.

• Si le champ obtenu est trop petit pour contenir l'ensemble de ses valeurs, ces dernières sont tronquées.

Taille maximale : Les règles de taille sont limitées par cette valeur.

Préfixe des noms de jeux de données : Lorsqu'un fichier est ouvert, il est affecté à un nom de jeu de données composé d'un préfixe et d'un entier séquentiel.

• Si ce nom de jeu de données est déjà utilisé, il est affecté à nouveau à ce fichier et l'autre jeu de données est fermé. Les modifications non enregistrées sont perdues.

La commande ALTER TYPE permet d'apporter des modifications. Les variables concernées sont signalées dans la sortie de la commande.

Sauvegarder

Par défaut, les fichiers modifiés restent ouverts et ne sont pas enregistrés. La sous-boîte de dialogue Enregistrer offre une fonction de sauvegarde pour plus de facilité.

Nouvel enregistrement de fichier après ajustement : Cochez cette case pour réenregistrer les fichiers traités.

• Le jeu de données actif n'est enregistré que si un répertoire est spécifié ci-après.

• Si le fichier n'a pas été modifié, il n'est pas réenregistré.

Ecraser le fichier existant : Cochez cette case pour remplacer le fichier existant par la version modifiée.

Ajouter un suffixe à la racine du nom de fichier : Lorsque vous enregistrez un fichier, un suffixe peut être ajouté à la racine du nom de fichier. Par exemple, si vous saisissez _adjusted, le nom de fichier deviendra mydata_adjusted.sav

Fermer le fichier après enregistrement : Cochez cette case pour fermer automatiquement le fichier une fois qu'il a été traité. Ceci est contre-productif si le fichier n'est pas également enregistré. Toutefois, le fichier actif n'est jamais fermé.

Ecrire les fichiers dans le répertoire : Sélectionnez le répertoire dans lequel écrire les fichiers. Sinon, les fichiers sont enregistrés à l'emplacement d'origine.

• Le fichier actif n'est pas enregistré sur le disque si un répertoire n'est pas indiqué. Lors de l'enregistrement, le nom se présente au format dsname.sav, dsname étant le nom du jeu de données. Aucun suffixe n'est ajouté.

Fonctions supplémentaires

A l'aide de la syntaxe, vous pouvez utiliser plus de deux fichiers d'entrée ou jeux de données, et le fichier actif n'a pas besoin d'être inclus. La commande fonctionne également avec un seul fichier.

Si vous indiquez un répertoire et que le dernier niveau n'existe pas, il est créé par cette commande.

Exécutez
STATS ADJUST WIDTHS /HELP.
pour afficher la syntaxe dans son intégralité.

Remarque : La version d'origine de cette commande réenregistre les fichiers et ce, même s'ils n'ont pas été modifiés. Bien que ce comportement soit parfois pratique, il a été modifié afin de travailler plus efficacement avec un grand nombre de fichiers.

Conditions requises

Cette commande requiert Python Essentials et la commande d'extension STATS ADJUST WIDTHS, disponibles sur le site Web de la communauté SPSS (www.ibm.com/developerworks/spssdevcentral).
