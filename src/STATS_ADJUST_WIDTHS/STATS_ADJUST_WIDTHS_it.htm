Adattare le larghezze delle variabili stringa nei file

Quando si uniscono i file che contengono le variabili stringa, le variabili corrispondenti devono avere la stessa larghezza. Questa procedura sincronizza le larghezze in una raccolta di file o dataset.

La finestra di dialogo opera nel file attivo più un file esterno. Utilizzando la sintassi, è possibile elaborare molti file in una volta.

►Per eseguire questa procedura, dai menu scegliere:

 Dati
  Dati>Adatta larghezze variabile...

File da sincronizzare Selezionare un file da sincronizzare con il file attivo.

• Se il file è già aperto, verrà riaperto e tutte le modifiche non salvate andranno perse.

• Il file attivo deve avere un nome dataset per eseguire questo comando.

Variabili da adattare Selezionare le variabili (stringa) di cui devono essere regolate le larghezze

• Una variabile selezionata deve comparire in ciascun file.

Regola di adattamento dimensione Scegliere il modo in cui le larghezze devono essere adattate. L'adattamento viene effettuato separatamente per ogni variabile. Per l'opzione Massimo, le larghezze delle variabili verranno adattate alla più ampia istanza della variabile, e così via. Se si sceglie Primo, la larghezza di destinazione sarà quella della variabile nel dataset attivo.

• Se il campo risultante è troppo piccolo per contenere tutti i valori, i valori verranno troncati.

Dimensione massima Le regole di dimensione sono costruite in base a questa dimensione massima specificata.

Prefisso per i nomi dataset Quando un file viene aperto, gli viene assegnato un nome dataset composto dal prefisso e un numero intero sequenziale.

• Se tale nome dataset è già in uso, viene riassegnato a questo file e l'altro dataset viene chiuso. Le modifiche non precedentemente salvate vengono perse.

Il comando ALTER TYPE viene utilizzato per apportare le modifiche. Le variabili interessate verranno notificate nell'output di tale comando.

Salva

Per impostazione predefinita, i file modificati vengono lasciati aperti e non salvati. La finestra secondaria Salva, fornisce funzioni di salvataggio per comodità.

Risalva file dopo adattamenti in caso di modifica Selezionare questa casella per salvare i file elaborati.

• Il dataset attivo non verrà salvato se non viene specificata una cartella di seguito.

• Se il file non è stato modificato, non viene risalvato.

Sovrascrivi file esistente Selezionare questa casella per sovrascrivere un file esistente con la versione modificata.

Accoda suffisso a radice nome file Quando si salva un file, può essere aggiunto un suffisso alla radice del nome file, ad esempio, specificando _adattato si otterrà il nome file dati_adattato.sav

Chiudi il file dopo il salvataggio Selezionare questa casella per chiudere automaticamente il file dopo che è stato elaborato. Questo è controproducente se il file non viene anche salvato, tuttavia il file attivo non viene mai chiuso.

Scrivi file nella cartella Selezionare una cartella in cui i file devono essere scritti. Se non è specificato nulla, i file vengono salvati nell'ubicazione originale.

• Il file attivo non viene salvato sul disco a meno che non venga specificata una cartella. Quando viene salvato, il nome ha il formato dsname.sav, dove dsname è il nome del dataset. Non vie ne accodato alcun suffisso.

Funzioni aggiuntive

Utilizzando la sintassi, è possibile lavorare con più di due file o dataset di input, senza dover includere il file attivo. Il comando funzionerà anche con un solo file.

Se si specifica una cartella, e l'ultimo livello non esiste, verrà creato da questo comando.

Eseguire
STATS ADJUST WIDTHS /HELP.
per visualizzare la sintassi completa.

Nota: la versione originale di questo comando risalva i file anche se non sono stati modificati. Nonostante talvolta conveniente, questo comportamento è stato modificato per migliorare le operazioni con un numero molto elevato di file.

Requisiti

Questo comando richiede Python Essentials e il comando di estensione STATS ADJUST WIDTHS presenti sul sito web SPSS Community (www.ibm.com/developerworks/spssdevcentral).
